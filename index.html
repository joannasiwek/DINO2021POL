git init - To polecenie stworzy nowy podkatalog o nazwie .git, zawierający wszystkie niezbędne pliki — szkielet repozytorium Gita.

git config --global user.name "" - polecenie ustawia nazwę użytkownika (--global oznacza, ze konfiguracja jest dokonywana jednorazowo) Anna Śmigiel

git git config --global user.email "" - polecenie ustawia adres email użytkownika (--global oznacza, że konfiguracje wystarczy dokonać jednorazowo) Maksymilian Kulikowski

git clone /path/to/repository - polecenie tworzące kopię roboczą lokalnego repozytorium

git checkout -b "branch" - polecenie tworzy nowego brancha o podanej nazwie i przenosi nas do niego. Filip Nowicki

git pull "remote" "branch" - polecenie zaciaga zmiany z repozytorium, aby lokalna wersja repozytorium byla aktualna, gdzie remote to nazwa remote'a (np, origin) a branch to nzawa brancha (np. master)

git clone /path/to/repository - polecenie klonuje zdalne repozytorium ze sciezki /path/to/repository do lokalnego folderu na naszym komputerze

git push "remote" "branch" - komenda ta spowoduje, że wszystkie zapisane zmiany znajdujące się na gałęzi "branch" zostaną wysłane na gałąź "branch" na zdalnym repozytorium ("remote" - nazwa repozytorium). Aleksandra Zdybał

git diff --staged - polecenie to pokaze zmiany dokonane jedynie w plikach "staged", tzn w plikach ktore sa przygotowywane na commit

git branch -d "branch" - polecenie usuwa branch o nazwie "branch"

git fetch "remote" "branch" - ta komenda pobiera zmiany z konkretnego brancha (np. main) do naszego lokalnego repozytorium z remote (np. origin czyli np. link do repo na github). Ale co jest ważne, zostanie aktualniona tylko historia zmian git czyli np. commity, ale zmiany nie zostaną zmergowane do tego brancha jak w przypadku pulla.

git config --global color.ui true - sprawia, że każdy output wpisany do terminala w gicie zostaje pokolorowany - Gabriela Piekarska

git commit "filename"- polecenie dodaje commit z naniesionaymi na plik zmianami.

git reset HEAD - polecenie to resetuje ustawienia HEADa do wskazanej pozycji

git branch -v -> polecenie branch z flagą verbose, wypisuje dostępne lokalnie branche, ubogacając tę informację o hash brancha, wiadomośc z ostatniego commita oraz upstream branch w przypadku trybu -vv (double verbose)

git branch --merged - wypisuję gałęzie połączone (zmergowane) z HEAD (czubkiem aktualnej gałęzi)

gitk -all - wyswietla zmiany w repozytorium. Obejmuje to wizualizacje wykresu zatwierdzenia, pokazanie informacji zwiazanych c kazdym zadtwierdzeniem oraz plikow w drzewach kazdej wersji. Flaga -all pokazuje wszystkie referencje

git bisect (start, bad, good, reset) - wykonuje binarny poszuk w commitach i sprawdza, czy istnieje regresja albo nie. Na kazdym etapie probuje zmniejszyc liczbe potencjalnie niebezpiecznych wersji o polowe. bad pokazuje czy istnije regresja, good odwrotnie. reset powraca do pierwotnego stanu przed uzyciem git bisect start. Aliaksei Konyshau.

git log --graph --all - polecenie wyswietla w terminalie caly log commitow i merge w formie drzewa, od pierwszego, zawiera taką informację, jak author commita , data , numer(hash), pomaga zrozumiec, gdzie potrzebny commit, w jakim branchu i kiedy był zrobiony∑

git branch "branch" "commit hash" - polecenie tworzy nowy branch o punkcie startowym we wskazanym commicie
